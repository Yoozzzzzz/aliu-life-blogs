## 1. 引言

### 1.1 编写目的

本文档在 `README.md` 产品说明基础上，将「静渊」音乐应用的前端范围、功能需求、界面与交互规范、性能与安全相关的前端职责进行可开发与可验收的细化，作为前端设计、实现、联调与测试的依据。

### 1.2 目标读者

前端开发、UI/UX 设计、测试、产品经理。

### 1.3 术语说明

| 术语 | 含义 |
|------|------|
| 极致纯净模式 | 无广告、无短视频、无开屏广告的内容与启动体验 |
| 私密歌单 | 默认仅用户可见、不参与系统/对外推荐的歌单 |
| 阅后即焚 | 退出应用后自动清除本地播放记录等敏感历史（可选保留媒体缓存） |
| 黑胶唱片库 | 面向 Hi-Res 等高品质音源与独立/小众曲风的曲库呈现 |

## 2. 目标与前端范围

### 2.1 产品目标（前端承接）

- 为目标用户（20–28 岁高压群体）提供**无干扰、高品质、精准推荐**的极简听歌界面。
- 前端需保障**默认可深色沉浸式**，并提供可选**日间天蓝**浅色主题；同时保障**手势优先**、**低操作路径**（启动即听、播放器极简），与产品「精神洁癖、私密治愈」定位一致。

### 2.2 前端范围界定

**包含：**

- 客户端内所有页面、组件、路由与状态管理（播放、歌单、推荐、设置等）。
- 与本地存储、离线缓存、音质档位展示、环境光/亮度相关的前端逻辑（在平台 API 允许范围内）。
- 与后端/云同步接口的对接（若存在）：默认本地优先，云端同步为用户显式开启后的能力。

**不包含（若由原生或单独团队负责需在此说明）：**

- 音频编解码、驱动级蓝牙编解码协商等非 Web 能力（按实际技术栈：Web / React Native / Flutter 等拆分模块责任）。

### 2.3 假设与依赖

- 存在可用的音源与元数据服务（或本地资源索引）；前端负责列表、播放 control、缓存策略的呈现与触发。
- 设计稿与组件库在「需求确认」阶段冻结初版，与本文档第 6 章一致。

## 3. 用户画像与关键场景

| 画像要点 | 前端体现 |
|----------|----------|
| 审美挑剔、反感低质内容与强推 | 默认弱化评论区、信息流；主界面信息密度低 |
| 深夜疗愈、抗焦虑 | 深色主题、可选深空黑/深海蓝；启动直达最近播放/助眠频道 |
| 隐私敏感 | 私密歌单默认态、阅后即焚设置与明确文案提示 |

## 4. 功能需求（前端）

以下按 `README` 模块分解为**可实现的界面与行为要求**。优先级与 README 一致。

### 4.1 极致纯净模式

| 编号 | 需求描述 | 验收要点 |
|------|----------|----------|
| F1.1 | 应用内不展示开屏广告、Banner 广告、短视频 Feed、直播入口等与「听歌」无关的强干扰模块 | 主干路径页面审计通过 |
| F1.2 | 冷启动后优先进入「可立即播放」的状态：展示最近播放或助眠频道入口，减少点击深度 | 自启动到可播放 ≤ 约定路径步数（与性能指标联动） |
| F1.3 | 「最近播放 / 助眠频道」数据来源与空态、加载态有明确 UI（骨架或轻量占位） | 无白屏、无未处理报错 |

### 4.2 私密歌单系统

| 编号 | 需求描述 | 验收要点 |
|------|----------|----------|
| F2.1 | 新建/默认歌单为**私密**；界面需展示私密标识，且符合「不被推荐」的产品表述（若与后端联动则展示一致状态） | 创建后默认私密；设置可切换（若产品允许） |
| F2.2 | 「阅后即焚」开关：开启后，**退出 App** 时清除本地播放历史（或等价「会话历史」）；可选「仍保留已缓存媒体文件」 | 开关状态持久化；退出重进后历史与文档一致 |
| F2.3 | 云端同步为**默认关闭**，需用户主动开启；设置页文案清晰说明数据范围 | 首次使用无_implicit_ 同步 |

### 4.3 精准音乐发现

| 编号 | 需求描述 | 验收要点 |
|------|----------|----------|
| F3.1 | 情绪/场景化推荐入口（如 #考研焦虑 #失眠）：标签可选、可组合或单选，与推荐列表联动 | 切换标签后列表更新；有加载/空态 |
| F3.2 | 「黑胶唱片库」或同级的 Hi-Res/品质曲库专区的列表与详情页：突出音质与曲风（独立、后摇、Lo-Fi 等） | 列表展示品质标识；进入播放流程正常 |
| F3.3 | 推荐与曲库页遵守纯净模式：无广告穿插 | 与 F1.1 一致 |

### 4.4 沉浸式交互

| 编号 | 需求描述 | 验收要点 |
|------|----------|----------|
| F4.1 | **主题**：用户可在 **深空黑 `#121212`**、**深海蓝 `#0D1B2A`**、**日间天蓝**（浅底 `#F0F9FF` / `#E0F2FE` 渐变，主强调 `#3B82F6`，辅强调 `#60A5FA`，正文 `#0F172A`）之间切换；支持**跟随系统**：系统为浅色外观时使用日间天蓝；系统为深色外观时使用用户所选深色主题（若用户仅选日间天蓝则深色外观回退为深空黑）。与 README「夜间模式不强制」一致。 | 主要页面用色符合第 6.1 节色板；跟随系统行为与设置说明一致 |
| F4.2 | 评论区**默认折叠**；提供「仅看歌词」或等价主视图，减少评论区对情绪的干扰 | 默认进入为歌词/折叠态；可展开 |
| F4.3 | **手势**：左滑返回、上滑收起播放器等与系统/导航栈一致的行为 | 核心页面完成手势验收清单 |
| F4.4 | **迷你/full 播放器**：默认仅 **封面 + 进度条 + 播放/暂停**；收藏、分享等通过**长按封面**等方式二次露出 | 默认视图无多余主按钮 |
| F4.5 | 若平台支持：**根据环境光调节建议亮度或系统亮度提示**（不替代系统权限模型） | 有则验证；不支持则降级为手动亮度说明 |

### 4.5 音质与设备（前端展示与控制）

| 编号 | 需求描述 | 验收要点 |
|------|----------|----------|
| F5.1 | 展示当前音质档位（如 Hi-Res ≥96kHz/24bit、AAC 等），与播放器实际策略一致 | 切换蓝牙/外放时 UI 与说明一致 |
| F5.2 | 在不可用 Hi-Res 的设备或蓝牙链路上，界面给予合理预期（降级说明，避免用户误解「无损未生效」） | 文案与状态一致 |

## 5. 非功能性需求（前端）

### 5.1 性能

| 指标 | 要求 | 前端实现要点 |
|------|------|----------------|
| 启动 | ≤1 秒进入可交互首屏（无开屏广告） | 路由懒加载策略、首屏数据缓存、避免阻塞渲染的大量同步脚本 |
| 播放起播 | 音乐加载延迟 ≤200ms（在缓存命中或本地场景下） | 预加载、离线优先、缓冲进度与降级提示 |
| 流畅性 | 列表滚动、切歌动画无明显掉帧 | 虚拟列表、图片/封面尺寸与缓存 |

### 5.2 安全与隐私（前端职责）

- 歌单、播放历史等**默认仅存本地**；仅在用户开启「云端同步」后，才按接口上传约定字段。
- 阅后即焚：退出时前端/壳层需执行清理本地历史记录的约定逻辑；异常需可恢复或记录（按产品是否要求日志）。
- 公共页面不展示用户私密歌单内容；分享功能若存在，需二次确认且默认不包含私密资源。

### 5.3 可访问性与可调参

- 字号层级：**标题 18pt、正文 14pt、辅助 12pt**，行距 **1.5 倍**（见第 6 章）；若系统字号缩放，布局不严重错位。
- 触控热区符合平台最小点击区域建议。

### 5.4 兼容与降级

- 不支持环境光传感器时，隐藏或降级相关 copy，不设死入口。
- 不支持 Hi-Res 解码的环境：隐藏或灰显 Hi-Res 仅作「内容标签」展示时与产品确认口径。

## 6. UI/UX 设计规范（摘录与落地）

### 6.1 色彩

**深色主题（深空黑 / 深海蓝）**

| 用途 | 色值 |
|------|------|
| 主背景（深空黑） | `#121212` |
| 主背景（深海蓝） | `#0D1B2A` |
| 辅助高亮（低饱和青） | `#4ECDC4` |
| 辅助高亮（淡紫） | `#E6E6FA` |

**日间天蓝主题（可选，对齐设计研习报告中的演示色板）**

| 用途 | 色值 |
|------|------|
| 页面浅幕渐变 | `#F0F9FF` → `#E0F2FE`（实现可为线性渐变） |
| 主强调（链接、主按钮、选中态） | `#3B82F6` |
| 辅强调 | `#60A5FA` |
| 正文主色 | `#0F172A` |
| 次要与弱文案 | `rgba(15,23,42,0.72)`、`#64748B` |
| 表面/卡片半透 | `rgba(255,255,255,0.85)` 等（以工程令牌为准） |

原则：深色模式下高亮色**仅用于按钮、关键状态**，避免大色块污染「安静」氛围；日间主题同样避免用大色块铺满遮挡内容，主强调色克制用于交互焦点。

### 6.2 字体与排版

- 字体：**无衬线**（如 PingFang SC）。
- 字号：**18 / 14 / 12** pt 三级；行距 **1.5**。

### 6.3 图标

- **线性图标**，**2px 描边**；避免厚重面性图标。

### 6.4 交互原则摘要

- 操作以**手势**为主，减少常驻按钮数量。
- 播放器**极简** + **长按封面**展开扩展操作。
- **主题**：深色与日间天蓝可选；**夜间模式**不强制，可跟随系统（浅色外观 ↔ 日间天蓝，深色外观 ↔ 用户所选深色或回退规则见 F4.1）。

## 8. 验收与测试关注点

- **功能**：F1.x–F5.x 用例全覆盖；阅后即焚、默认私密、默认不同步的边界场景。
- **性能**：冷启动、列表首屏、弱网与离线切换。
- **视觉**：抽样对照设计稿与色板、字号、图标风格。
- **纯净**：任意一级页面不出现README 明确禁止的广告/短视频/直播入口。

## 10. 风险（前端相关）

| 风险 | 应对 |
|------|------|
| 性能指标与真实设备差异大 | 早中期在低端机与蓝牙场景实测；定义「可交互首屏」的统一定义 |
